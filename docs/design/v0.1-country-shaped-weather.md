### 1. Country IP-space weather (right panel)

**Implementation:** Render the compact national Hilbert (`COMPACT_SIZE` canvas) as an SVG `<image>` element, masked with an SVG `<clipPath>` of the target country polygon.

```svg
<defs>
  <clipPath id="country-mask">
    <!-- Cuba GeoJSON polygon → SVG path -->
    <path d="M x1 y1 L x2 y2 ... Z" />
  </clipPath>
</defs>
<image href="hilbert-canvas.png" clip-path="url(#country-mask)" />
```

Alternatively, render the Hilbert directly into a canvas, then use `ctx.clip(countryPolygon)` to draw. Either way, the Hilbert texture is unchanged — we just show the parts inside the country.

**Pros:**
- Hilbert cell positions preserved (IP locality ordering intact)
- No new data pipeline needed
- Country shape is immediately recognizable
- "Weather inside the country" visual metaphor

**Hit testing:** Maintain a separate pixel→cell lookup map (already used by the Hilbert canvas for hover). When a click falls inside the country polygon, map to the nearest Hilbert cell. Same interaction model as current Hilbert.
- Get click coordinates
- Apply the same inverse transform (screen coord → Hilbert cell index)
- If cell exists and falls inside the country polygon, select it
- If click is outside the polygon, ignore

**Prefix-to-shape mapping stability:** The current `compact.ts` already maps every Cuban prefix to a Hilbert cell. The mapping is deterministic and stable. The only change is visual — we're clipping the rendering area, not changing the data structure.

### 2. Geographic collector map (left panel)

**Implementation:** Simple SVG map (no deck.gl). Show:
- Faint landmass outlines (North America, Europe, Caribbean)
- Cuba polygon highlighted
- Collector points at their documented locations
- City labels
- Curved connection lines from collectors → Cuba center (low opacity, decorative)

**Data sources:**
- Collector locations: `config/collectors.json` (lat/lon → SVG projection)
- Cuba polygon: existing `cubaGeoJson` in `geo.ts`
- Landmasses: existing `landmasses` in `geo.ts`

**Projection:** Simple equirectangular (Plate Carrée). Good enough for a regional view of North America/Europe at this scale. No projection library needed — just scale lat/lon to SVG coordinates:

```ts
function project(lon: number, lat: number, w: number, h: number): [number, number] {
  const x = ((lon + 180) / 360) * w;
  const y = ((90 - lat) / 180) * h;
  return [x, y];
}
```

**Complexity:** Low. SVG with simple projection. All data already available.

### 3. Transit ASNs (middle panel)

Adapt the existing PathGraph Sankey but:
- Left column = collector node positions (from the geographic map)
- Middle column = transit ASN nodes (same as current)
- Right column = flows entering the country shape

Curves connect collectors → transit ASNs → country. Transit nodes are labeled with ASN names from the catalog.

**Flow thickness** = number of path families using that transit ASN.

### 4. Combined interaction

- **Click a collector** on the map → highlight that collector in the Sankey, filter path graph, update side panel
- **Click a transit ASN** → highlight path families using that transit, dim others
- **Click inside the country shape** → hit-test to find the nearest prefix cell → select prefix, highlight path families that reach it, update side panel

### 5. Hilbert inset

The original Hilbert square can be rendered as a small technical inset below the country shape, labeled "IP-space fingerprint (Hilbert curve)." This preserves the technical explanation while letting the country shape carry the visual story.

### 6. Implementation complexity

| Component | Approach | Complexity | Risk |
|-----------|----------|:---:|------|
| Country-shaped weather | SVG clipPath masking Hilbert canvas | Low | Clip path generation from GeoJSON |
| Geographic collector map | SVG equirectangular projection | Low | Simple math, existing data |
| Transit flows | Adapt existing PathGraph SVG | Low | Minor layout changes |
| Hit testing in country shape | Existing pixel→cell lookup + point-in-polygon | Medium | Must handle edge cells (partially in/out) |
| Combined layout | CSS grid on existing container | Low | Replace main content flex layout |

**Total: ~200-300 lines of new/modified code.** No new dependencies. No pipeline changes.

### 7. Migration path from current layout

1. **Phase A** (this milestone): Replace the center PathGraph with the combined three-panel layout. The PathGraph code gets absorbed into the transit ASN column.
2. **Phase B** (later): Move the Hilbert inset to a small technical reference panel.
3. **Phase C** (future): Add traffic signal as a data-plane overlay texture on the country shape.

### 8. Why not the other options

**Option A (clipped Hilbert only):** Would work for the country shape but loses the collector geography and transit flows that make the AS-path story.

**Option B (packed tiles):** Requires a packing algorithm that doesn't preserve the Hilbert ordering (IP locality). The tiles would be purely decorative cells, losing the address-space meaning.

**Option C with clipped Hilbert** preserves the technical IP-space mapping AND adds the geographic/AS-path context.

### 9. Country shape generation

The Cuba GeoJSON polygon needs to be converted to an SVG path. This can be done once at build time or at load time:

```ts
function geoJsonToSvgPath(feature: Feature<Polygon>, w: number, h: number): string {
  return feature.geometry.coordinates[0]
    .map(([lon, lat], i) => `${i === 0 ? 'M' : 'L'} ${x(lon, w)} ${y(lat, h)}`)
    .join(' ') + ' Z';
}
```

The polygon is already in `geo.ts` as `cubaGeoJson`. Same data, just rendered as SVG path.

### Recommendation summary

**Implement Option C.** Country-shaped weather via SVG-clipped Hilbert + geographic collector map + transit flows. No new dependencies. ~300 lines of code. Stable hit testing via existing pixel→cell lookup. All current functionality preserved.
